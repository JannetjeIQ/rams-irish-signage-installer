# Permits-to-Work Framework

When a permit is needed, who issues it, and how the specialist handles it on the RAMS.

A permit-to-work is a written authorisation for a specific high-risk task, signed by a competent person, valid for a defined window. On most jobs the main contractor or PSCS issues the permit and the install contractor complies. When the install contractor is itself the PSCS, it issues the permit.

## When each permit fires

| Permit | Triggered when |
|--------|----------------|
| **Hot Work Permit** | Use of heat guns, grinders that throw sparks, soldering, or any open flame on or near combustible material |
| **Work at Height Permit** | Height work over 2m without collective fall protection, or where the main contractor requires it for any height work regardless |
| **Confined Space Permit** | Entry into any confined space: ceiling voids, lift shafts, plant rooms, tanks. Rare for signage, possible for some wayfinding installs |
| **Out-of-Hours Permit** | Work outside normal site operating hours; almost always required on healthcare, transport, retail estates |
| **Lone Working Permit** | Single-fitter work in isolated or unattended premises |

## Standard practice when a permit applies

1. Request the permit from the main contractor or site manager at induction stage (or earlier for advance scheduling).
2. Permit signed and dated **before** work begins.
3. Permit cancelled and signed off at end of shift.
4. Copy of the permit attached to the RAMS as Appendix D.
5. Each fresh shift requires a fresh permit unless the permit explicitly covers a multi-day window.

## Specialist behaviour

When the user describes a job that triggers any of the above permits, the specialist will:

- Surface the permit requirement in the gap-questions phase ("Hot work permit needed: is this in place?").
- Add a Method-of-Work step that says "Confirm permit signed before commencing the [permit-type] task".
- Add the permit to the Appendix D index in Section 8.
- Add a hold-point in the install steps if the permit is mid-sequence (e.g., heat-gun work after panel placement).

## When the contractor is the PSCS

If the contractor profile says or the job confirms that the install contractor is itself the PSCS for the works, the specialist will additionally:

- Flag that the permit must be issued by the contractor's responsible person, not requested from someone else.
- Recommend a permit register for the duration of the works.
- Recommend a copy of the permit lodged with the client at handover.

## Notes

- A permit-to-work is not a substitute for the controls in the RAMS. The RAMS still has to demonstrate that the work can be done safely; the permit is the moment-of-execution authorisation that the conditions on the day support that demonstration.
- Permit forms are often site-specific. Where the main contractor has a standard form, use theirs. Where they do not, the specialist can produce a generic form on request.
